# AGENTS.md — ESP32_AquaEdge

## Build & Run
- **Toolchain**: PlatformIO (`pio`). Do not use raw `make` or `idf.py`.
- **Common commands**:
  - `pio run` — build
  - `pio run --target upload` — flash to board
  - `pio test` — run tests (currently none implemented)
  - `pio run --target clean` — wipe build artifacts
- **Board / env**: `esp32dev` (see `platformio.ini`). Single env, no multi-target setup.
- **Framework**: Arduino on ESP32 (not ESP-IDF native).

## Architecture (non-obvious)
- **Entry point**: `src/main.cpp` wires GPIO pins and instantiates `IrrigationController`.
- **Main loop**: fully non-blocking. `IrrigationController::tick()` runs on a 5-second `millis()` interval.
- **Layered flow**:
  1. Sensors (`read()`) → DTOs (`Readings.h`)
  2. `AgronomicEvaluator::evaluate()` → `AgronomicDiagnosis`
  3. Actuators (`execute()`) driven by diagnosis
  4. `TelemetryClient::send()` logs to Serial (115200 baud)
- **Critical safety rule**: if `WaterLevelReading.status == EMPTY`, evaluator immediately disables both pumps and short-circuits. Any new logic must preserve this behavior.
- **Class diagram**: `docs/class-diagram.puml` (PlantUML). Note: diagram names are in Spanish but code uses English identifiers.

## Adding Components
- **New sensor**: inherit from `BaseSensor`, add reading struct to `Readings.h`, add `read()` method returning the DTO. Update `IrrigationController` composition.
- **New actuator**: inherit from `BaseActuator`, add command to `Command` enum in `Types.h`, implement `execute()`. Update `IrrigationController` composition.
- **Evaluator changes**: thresholds are hardcoded in `AgronomicEvaluator` (e.g., `MOISTURE_THRESHOLD = 30.0f`).

## File & Code Conventions
- **Line endings**: `.gitattributes` forces LF for `*.cpp`, `*.h`, `*.ino`, `*.json`, `platformio.ini`. Do not introduce CRLF.
- **Language**: identifiers and comments are English (even though some historical comments are Spanish). Keep English for all new code.
- **Headers**: use `#ifndef` guards. Follow the existing `include/` + `src/` split.
- **Secrets**: `.env`, `secrets.h`, and `.env.*` are gitignored. Never commit WiFi credentials or API keys.

## VS Code / IDE — Do Not Touch
- `.vscode/c_cpp_properties.json` and `.vscode/launch.json` are **auto-generated by PlatformIO**. If IntelliSense or debug paths are wrong, fix them via `platformio.ini` (build flags / include paths) and regenerate, never by hand-editing those JSON files.

## Hardware Notes
- **GPIO mapping** is centralized in `src/main.cpp`. Do not scatter pin numbers across classes.
- **Current assignments** (see also `docs/hardware.md`): water pump GPIO 14, fertilizer pump GPIO 13. Both are safe outputs (not strapping pins). GPIO 12 was previously used but has been moved because it is a strapping pin.
- **ADC calibration constants** live in sensor headers (`AIR_VALUE`, `WATER_VALUE`, `MAX_RESISTANCE`, `MIN_RESISTANCE`). These are device-specific and require physical calibration.
- **Serial**: `Serial.begin(115200)` is the only diagnostic output channel right now.
